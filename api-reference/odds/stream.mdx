---
title: "Get a real-time Server-Sent Events stream of odds updates"
api: "GET /odds/stream"
description: "Get a real-time Server-Sent Events stream of odds updates"
---

A long-lived stream of odds updates.

The stream consists of two types of events:
1. **Data messages:** These have an `id` in the format `timestamp:index` and `data` containing a JSON object for a single Odds record.
2. **Heartbeats:** These have an `event` field set to `heartbeat` and may have data like `{"Ts": 12345}`.

## Query Parameters

<ParamField query="fixtureId" type="int64">
  Optional. Filter the event stream for a single fixture ID.
</ParamField>

## Headers

<ParamField header="Last-Event-ID" type="string">
  Optional. The ID of the last event received, to resume the stream.
</ParamField>

## Response

Returns a Server-Sent Events (SSE) stream with content type `text/event-stream`.

### Data Events

Each data event contains:
- **id**: Event identifier in format `timestamp:index`
- **data**: JSON object containing an Odds record

### Odds Object Structure

<ResponseField name="FixtureId" type="int64" required>
  Fixture identifier
</ResponseField>

<ResponseField name="MessageId" type="string" required>
  Unique message identifier for the odds update
</ResponseField>

<ResponseField name="Ts" type="int64" required>
  Timestamp of the odds update
</ResponseField>

<ResponseField name="Bookmaker" type="string" required>
  Bookmaker name
</ResponseField>

<ResponseField name="BookmakerId" type="int32" required>
  Bookmaker identifier
</ResponseField>

<ResponseField name="SuperOddsType" type="string" required>
  Type of odds market
</ResponseField>

<ResponseField name="InRunning" type="boolean" required>
  Indicates if the fixture is currently in play
</ResponseField>

<ResponseField name="PriceNames" type="array<string>">
  Names of the prices/outcomes
</ResponseField>

<ResponseField name="Prices" type="array<int32>">
  Odds values for each outcome
</ResponseField>

### Heartbeat Events

Heartbeat events keep the connection alive and contain:
- **event**: Set to `heartbeat`
- **data**: JSON object like `{"Ts": 12345}`

<CodeGroup>
```text Example Stream Events
id: 1704067200000:0
data: {"FixtureId":1001,"MessageId":"msg_12345","Ts":1704067200000,"Bookmaker":"BetMGM","BookmakerId":5,"SuperOddsType":"Moneyline","InRunning":false,"PriceNames":["Alabama","Georgia"],"Prices":[150,200]}

event: heartbeat
data: {"Ts": 1704067260000}
```
</CodeGroup>
