---
title: "Accept an existing, signed trade offer"
api: "POST /trading/accept"
description: "Accept an existing, signed trade offer"
---

Allows an authenticated user to accept an open offer. The payload must include the `offerId` and a valid Ed25519 `signature`.

The server will verify that the `signature` is a valid signature of the Borsh-serialized `offerId`, signed by the `acceptingTraderPubkey`. This cryptographically proves the user's intent to accept the trade.

The server also soft-checks that the token balance available in the user token account is sufficient to cover their accepting stake.

If everything is valid, the endpoint immediately returns a `202 Accepted` status and kicks off the asynchronous, on-chain `create_trade` signing protocol.

## Request Body

The ID of the offer being accepted, the acceptor's public key, and their Base58-encoded signature.

<ParamField body="offerId" type="int32" required>
  The ID of the offer to accept
</ParamField>

<ParamField body="acceptingTraderPubkey" type="string" required>
  Public key of the trader accepting the offer
</ParamField>

<ParamField body="signature" type="string" required>
  Base58-encoded Ed25519 signature of the Borsh-serialized offerId
</ParamField>

## Response

Returns HTTP status `202 Accepted` with a confirmation message that the acceptance was valid and the on-chain signing process has begun.

<ResponseField name="message" type="string">
  Confirmation that the acceptance was valid and the on-chain signing process has begun
</ResponseField>
