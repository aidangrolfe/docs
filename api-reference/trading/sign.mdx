---
title: "Submit a signature for a trade"
api: "POST /trading/sign"
description: "Submit a signature for a trade"
---

Allows an authenticated user to submit their signature for a previously prepared multi-signer trade.
The server will verify that the `signer` public key in the payload matches the authenticated user's identity.

This endpoint has two possible success responses:
1. **202 Accepted**: If the submitted signature is valid but the trade is still awaiting other participants' signatures.
2. **200 OK**: If this is the final signature required. The service will submit the fully-signed transaction to the network and the response body will contain the on-chain transaction signature.

## Request Body

The trade ID, the public key of the signer, and their Base58-encoded signature of the transaction message.

<ParamField body="tradeId" type="string" required>
  The ID of the trade being signed
</ParamField>

<ParamField body="signer" type="string" required>
  Public key of the signer
</ParamField>

<ParamField body="signature" type="string" required>
  Base58-encoded Ed25519 signature of the transaction message
</ParamField>

## Response

### 200 OK Response

Final signature received. The transaction has been submitted to the network.

<ResponseField name="transactionSignature" type="string">
  The on-chain transaction signature
</ResponseField>

### 202 Accepted Response

Signature was successfully received and stored. Awaiting signatures from other participants.

<ResponseField name="message" type="string">
  Confirmation that the signature was received and the trade is awaiting additional signatures
</ResponseField>
